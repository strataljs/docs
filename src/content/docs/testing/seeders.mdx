Seeders populate your test database with predictable data before each test run. The `@stratal/testing` package provides a `Seeder` base class that receives the database service as a parameter and runs inside a transaction via `TestingModule.seed()`.

<Aside type="tip">
  Looking for application-layer seeders that run via the CLI? See [Application Seeders](/framework/seeders/).
</Aside>

## Creating a seeder

Extend the `Seeder` base class and implement the `run(db)` method:

```typescript
import { Seeder } from '@stratal/testing'
import type { DatabaseService } from '@stratal/framework/database'

export class UserSeeder extends Seeder {
  async run(db: DatabaseService): Promise<void> {
    await db.user.create({
      data: {
        id: 'test-user-id',
        email: 'user@test.com',
        name: 'Test User',
        emailVerified: true,
        role: 'user',
      },
    })
  }
}
```

The `run()` method receives the `DatabaseService` instance directly — no constructor injection needed. This keeps test seeders simple and self-contained.

## Running seeders

Use `TestingModule.seed()` to run one or more seeders. Each call runs inside a database transaction, so all operations either succeed or roll back together.

```typescript
import { TestingModule } from '@stratal/testing'
import { UserSeeder } from './seeders/user.seeder'

const module = await TestingModule.create(AppModule)

// Run a single seeder
await module.seed(new UserSeeder())

// Run multiple seeders in a single transaction
await module.seed(new UserSeeder(), new RoleSeeder())
```

## Named connections

The `Seeder` class supports generic connection names for multi-database setups:

```typescript
import { Seeder } from '@stratal/testing'
import type { DatabaseService } from '@stratal/framework/database'

export class AnalyticsSeeder extends Seeder<'analytics'> {
  async run(db: DatabaseService<'analytics'>): Promise<void> {
    await db.event.create({
      data: { type: 'page_view', path: '/' },
    })
  }
}
```

Pass the connection name as the first argument to `seed()`:

```typescript
await module.seed('analytics', new AnalyticsSeeder())
```

## Typical test pattern

A common pattern is to truncate the database and re-seed before each test:

```typescript
import { describe, beforeEach, afterAll, it } from 'vitest'
import { TestingModule } from '@stratal/testing'
import { UserSeeder, ADMIN_USER_ID } from './seeders/user.seeder'

describe('UsersController', () => {
  let module: TestingModule

  beforeEach(async () => {
    module = await TestingModule.create(AppModule)
    await module.truncateDb()
    await module.seed(new UserSeeder())
  })

  afterAll(async () => {
    await module.close()
  })

  it('returns the seeded user', async () => {
    await module.assertDatabaseHas('user', { id: ADMIN_USER_ID })
  })
})
```

## Full example

A more complete seeder that creates multiple related records:

```typescript
import { Seeder } from '@stratal/testing'
import type { DatabaseService } from '@stratal/framework/database'

export const ADMIN_USER_ID = 'admin-user-id'
export const REGULAR_USER_ID = 'regular-user-id'

export class UserSeeder extends Seeder {
  async run(db: DatabaseService): Promise<void> {
    const [admin, regular] = await Promise.all([
      db.user.create({
        data: {
          id: ADMIN_USER_ID,
          email: 'admin@test.com',
          name: 'Admin User',
          emailVerified: true,
          role: 'admin',
        },
      }),
      db.user.create({
        data: {
          id: REGULAR_USER_ID,
          email: 'user@test.com',
          name: 'Regular User',
          emailVerified: true,
          role: 'user',
        },
      }),
    ])

    await db.account.createMany({
      data: [
        {
          accountId: admin.id,
          providerId: 'credential',
          userId: admin.id,
          password: 'hashed-password',
        },
        {
          accountId: regular.id,
          providerId: 'credential',
          userId: regular.id,
          password: 'hashed-password',
        },
      ],
    })
  }
}
```
